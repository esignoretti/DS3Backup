    - `NewBackupJobRunner(cfg *config.Config, getJob func(string) *models.BackupJob, runBackupFn func(*models.BackupJob) (*models.BackupRun, error)) *BackupJobRunner`
    
    - `(r *BackupJobRunner) RunJob(jobID string)` ‚Äî Main execution entry point called by cron:
      1. Gets job via `getJob`
      2. If nil, log warning and return
      3. If !job.Enabled, log and return
      4. Calls `runBackupFn(job)`
      5. If run.Error != "", log as warning
      6. Updates `job.LastRun` and `job.LastError` in the config
      7. Calls `r.cfg.SaveConfig()` to persist updated timestamps
      8. On failure, don't let the panic propagate ‚Äî log and return
    
    - `(r *BackupJobRunner) RunJobAsync(jobID string)` ‚Äî Wraps RunJob in a goroutine with recover() for panic safety. Used as the cron callback factory.
    
    - `RunnerFactory() func(jobID string) func()` ‚Äî Returns a factory function: given jobID, returns a closure that calls `RunJobAsync(jobID)`. This is the `runnerFactory` parameter used by `Scheduler.ReloadJobs()`.
