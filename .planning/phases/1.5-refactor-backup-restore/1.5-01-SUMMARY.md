## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 2 - Missing type cast] AWS SDK typed enum for ObjectLockMode**
- **Found during:** Task 1
- **Issue:** The `ObjectLockMode` field in `s3.PutObjectInput` requires `s3types.ObjectLockMode` (a typed string), not a raw `string`. Using `mode` directly caused a compile error.
- **Fix:** Added `s3types "github.com/aws/aws-sdk-go-v2/service/s3/types"` import and cast to `s3types.ObjectLockMode(mode)`.
- **Files modified:** internal/s3client/client.go
- **Commit:** `a6ac1c4`

**2. [Rule 2 - Missing critical functionality] Config save on failure would persist stale LastRun**
- **Found during:** Task 1
- **Issue:** The plan identified this for Task 1 and included it. Implemented per D-08.
- **Fix:** Wrapped config save in `if err == nil` block.
- **Commit:** `a6ac1c4`

**3. [Rule 3 - badger/v3 already absent]**
- **Found during:** Task 2
- **Issue:** The plan described removing `github.com/dgraph-io/badger/v3 v3.2103.5` from go.mod, but badger/v3 was already absent from go.mod and go.sum. The dependency was likely removed in a previous commit or during a `go mod tidy` pass.
- **Action:** Confirmed clean state, ran `go mod tidy` (no changes), continued with remaining Task 2 items.
- **Commit:** `f69555f`
