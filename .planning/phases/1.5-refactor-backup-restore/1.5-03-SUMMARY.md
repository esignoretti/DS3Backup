# Phase 1.5 Plan 03: Restore Pipeline Refactor, Stub Cleanup & Index Rebuild

## One-liner

Eliminated ~118 lines of duplicate restore worker patterns by extracting a shared `runDownloaderPipeline` method, removed 45 lines of dead `RebuildEngine` stubs from `recovery/rebuild.go`, and replaced the TODO stub in `index rebuild` with a working implementation that scans S3 batch manifests and file objects to reconstruct the local BadgerDB index.

## Tasks Executed

### Task 1: Consolidate duplicate restore worker patterns into shared pipeline

- **Files:** internal/restore/engine.go
- **Actions:**
  - Extracted `runDownloaderPipeline(entries, opts, tracker)` — handles overwrite filtering, dry-run detection, job submission, and result collection
  - Two code paths: goroutine-based with mutex + ProgressTracker (for RestoreWithProgress, RestoreEntries) and synchronous select-based (for Restore)
  - Refactored `Restore()` from ~90 lines to ~25 lines (delegates to pipeline, sets Duration)
  - Refactored `RestoreWithProgress()` from ~110 lines to ~20 lines (delegates unconditionally)
  - Refactored `RestoreEntries()` from ~105 lines to ~15 lines (delegates unconditionally)
  - `ResumeRestore()` left unchanged (uses DownloaderV2 with FileState-based jobs)
  - Total engine.go reduced from 626 to 508 lines (~118 eliminated)
- **Verification:** `go build ./...` ✅, `go vet ./...` ✅

### Task 3: Implement index rebuild from S3 objects

- **Files:** internal/cli/index.go
- **Actions:**
  - Replaced the TODO stub (lines 144-149) with full rebuild logic
  - Lists all S3 objects under `backups/<jobID>/` prefix
  - Classifies objects: batch manifests (`*-manifest.json.enc`) and individual file objects (`/files/` prefix)
  - Processes batch manifests: downloads via `s3Client.GetObject()`, unmarshals JSON into `models.BatchManifest`, creates full `FileEntry` records with path, size, hash, batch offset, and backup time, saves via `idxDB.SaveEntry()`
  - Processes individual files: extracts hash hex from S3 key (`.../<hash>.enc`), creates placeholder `FileEntry` with `(unknown) - <hash>` path, saves to index
  - Added imports: `encoding/hex`, `encoding/json`, `log`, `strings`, `github.com/.../pkg/models`
  - Note: individual file objects are encrypted and cannot be decrypted without password — placeholders allow index-based restore operations
- **Verification:** `go build ./...` ✅, `go vet ./...` ✅
- **Commit:** `a18dfb1`

- **Found during:** Task 1

- **Fix:** The `runDownloaderPipeline` correctly handles the two code paths — tracker path uses goroutine + mutex, non-tracker path uses synchronous select-based collection. Both paths submit from the same filtered `filesToRestore` slice.
